Ideation

Wednesday 2nd September

I want to take the question aspect of Archive and bring that to life.

My idea is to code a multi-choice questionnaire, that users interact with. The idea is that the answers have 'hidden' attributes connected to them, and when an answer is selected, the 'hidden attribute' is then slotted into an end-screen card which displays information about the user.
To keep this realistic and within a reasonable skill range, I want to focus on solely multi-choice questions, and then have a final card at the end which features the basics. Name, Age, Likes, Dislikes, Fears. Features like the generated advertisements will be too difficult to do.
Creating a full-screen PC application which is accessed by individuals. Upon further consideration, I would like this project to be more gallery-based, a small group of individuals who have an interest in what they are doing. I feel that would work best.

There are two ways to approach this:

1) I would need to brush up on OOP JavaScript. This option allows for maximum creativity and customisation, but I am not very good at JavaScript, I would need to find multiple tutorials.

2) Use a third-party quiz builder, this option is much easier and more efficient but also might not have much room for customisation.

I would like to try a search for JavaScript tutorials first, but if I cannot find anything I will resort to option two.

Development

Monday 14th September

Upon further thinking, I had decided to develop two interactive aspects of Archive. These are the 'quiz' aspect and the database. This way, I can show visually what I am aiming for. Producing assets that users can interact with.

Quiz:

After much research, I found that I simply did not have the coding knowledge necessary in order to code a fully functional quiz application. I instead opted to use Adobe XD, to produce a prototype instead. The idea is that it serves as a visual representation of how the set-up and user flow would function.

Here is a screen-recording of it in action:

Here, I demonstrate how a user may navigate and decide their answers. Whilst these are not definitive final questions, they serve as a representation of the sudden privacy invasion I want my application to invoke. The questions start off seemly harmless and simple, but as the user progresses, they become much more personal.

I may add a final 'card' that comes up with a perceived identity at the end, although that is dependant on time and if I deem it truly necessary.

Coding

Monday 14th September

For privacy reasons, I have opted to code pre-set false personas, as opposed to creating an option to create new cards. But this feature may be added provided the right coding knowledge is found.

I have a combination of HTML, CSS, and JavaScript to create a snippet of what a database could look like. I created a carousel, which is essentially a slideshow. Using the Flickity plugin, I was able to create a fully interactive slideshow.
